# План дальнейших улучшений mexc_spread

Приоритеты: 🔴 — важно/быстро окупается, 🟡 — желательно, 🟢 — по мере надобности.

## 1. Надёжность и качество кода

### 🔴 1.1 Починить 21 падающий backend-тест
`tests/test_reconciliation.py`, `test_spread_capture_reconciliation.py`, `test_position_manager_reconciliation.py`, `test_execution_model.py`, `test_persistence.py` падают ещё до моих изменений (в основном `AttributeError: NEW` — похоже, enum состояния позиции переименован, а тесты не обновлены). Пока тесты красные, они не ловят реальные регрессии в торговой логике — это самый рискованный участок (реальные деньги).

### 🔴 1.2 TypeScript: 29 ошибок tsc, нет скриптов lint/typecheck
`npx tsc --noEmit` даёт 29 ошибок в 8 файлах (неиспользуемые импорты, `Property 'symbol' does not exist`, небезопасные касты в `TradingAdminModal`). Добавить в package.json скрипты `"typecheck": "tsc --noEmit"` и `"lint"` (eslint), починить ошибки, включить в CI.

### 🔴 1.3 CI (GitHub Actions)
В репозитории нет CI вообще. Один workflow: pytest + tsc + vitest + vite build на каждый PR. Без CI регрессии видны только «в бою».

### 🟡 1.4 Удалить/починить мёртвый DNS-механизм
`config/network_dns_config.json` + `CustomDNSHTTPTransport` реально не подключены к запросам (см. мой анализ ранее): резолвер вызывает системный `getaddrinfo`, подмена hostname ломает SNI. Либо удалить весь слой (~3 файла), либо реализовать честный DoH-резолвер с корректным SNI.

### 🟡 1.5 requirements: зафиксировать dev-зависимости
`pytest` и `hypothesis` не устанавливаются из requirements — тесты «из коробки» не запускаются. Добавить `requirements-dev.txt`.

## 2. Производительность (продолжение)

### 🔴 2.1 Расширить WS-фиды на остальные CEX
Сейчас живой стакан по WS есть у MEXC (futures), Binance (futures), dYdX. Добавить аналогичные фиды (архитектура `_Feed` в `mexc_monitor/ws_bookticker.py` готова, нужен только parser + URL + volume-loader на биржу):
- OKX: `wss://ws.okx.com:8443/ws/v5/public`, канал `tickers`
- Bybit: `wss://stream.bybit.com/v5/public/linear`, канал `tickers`
- Gate.io: `wss://fx-ws.gateio.ws/v4/ws/usdt`, канал `futures.book_ticker`
- Bitget, HTX — аналогично
Итог: любой снапшот < 50 мс, автообновление можно опустить до 2–3 с.

### 🟡 2.2 SSE/WebSocket от бэкенда к фронтенду
Сейчас фронт поллит `/api/snapshot`. Когда данные и так живут в памяти бэкенда, логичнее пушить дифы по SSE (эндпоинт `/api/stream` уже частично есть для истории) — цены в таблице будут обновляться в реальном времени без запросов.

### 🟡 2.3 Дифференцированный ответ снапшота
Отдавать только изменившиеся строки (по `If-None-Match` уже есть 304 на «ничего не изменилось»; следующий шаг — параметр `since` и отдача дельты). Для MEXC-снапшота из 2200 строк это сократит трафик в разы.

### 🟢 2.4 Виртуализация и мемоизация MultiExchangePage
Страница «Мультибиржа» опрашивает все биржи последовательно на клиенте — распараллелить fetch'и (Promise.all) и кэшировать между переключениями символа.

## 3. Торговая функциональность (заработок)

### 🔴 3.1 Алерты на спред/базис
Сигналы — главное для заработка: задать порог (например, cross-spread > 30 bps между биржами) и получать уведомление (Telegram-бот уже частично есть в `mexc_monitor/alerts`). Сейчас алерты не покрывают мультибиржевые спреды.

### 🟡 3.2 История спредов между биржами
`history_store` пишет только MEXC spot/futures. Записывать топ-N кросс-биржевых спредов, чтобы видеть, какие пары стабильно дают возможность, а какие — разовый шум.

### 🟡 3.3 Учёт комиссий и funding в «Мультибирже»
Кросс-спред сейчас грязный (bid/ask без комиссий). Добавить net-спред с taker-fee обеих ног и funding-ставками — как уже сделано для одиночных бирж (`net_spread_bps`).

### 🟡 3.4 Калькулятор сделки
Клик по строке → мини-форма: объём → ожидаемый PnL с учётом комиссий, слippage по стакану (глубина уже доступна через `/api/orderbook`).

### 🟢 3.5 Paper-trading журнал
Кнопка «зафиксировать возможность» → журнал виртуальных сделок с последующей сверкой, закрылся ли спред. Помогает калибровать пороги без риска.

## 4. Безопасность и эксплуатация

### 🔴 4.1 ADMIN_TOKEN по умолчанию пустой
Если `ADMIN_TOKEN` не задан, `_require_admin_token` пропускает всех — торговые эндпоинты (`/api/trading/*`) открыты. Сделать торговлю недоступной без явно заданного токена.

### 🟡 4.2 Логирование в файл + ротация
Сейчас всё в stderr. Настроить `logging` с ротацией и отдельным уровнем для WS-фидов, иначе диагностика проблем (как с гео-блоками) затруднена.

### 🟡 4.3 Dockerfile / docker-compose
Один `docker compose up`: backend + собранный фронт (nginx). Убирает проблемы «у меня локально не запускается» и расхождения окружений.

### 🟢 4.4 Healthcheck WS-фидов
`/api/health` расширить статусом фидов (подключён/сколько символов/возраст последнего сообщения) — сразу видно, какая биржа отвалилась.

## 5. UX (мелочи, оставшиеся с прошлого обзора)
